웹 페이지
 - 정적페이지 (Static) : HTML, CSS, JS(JavaScript), Image
    데이터베이스에서 정보를 가져오거나 별도의 서버에서의 처리가 없어도
    사용자들에게 보여 줄수 있는 페이지
    어떠한 사용자가 오던간에 동일한 페이지를 보여준다
    
 - 동적페이지 (Dynamic) : JSP, 
    서버에서 데이터베이스에서 정보를 가져와서 처리하는 것처럼, 어떠한 요청에 의하여 
    서버가 일을 수행하고 해당 결과가 포함된 파일을 보여주는 페이지
    사용자들마다 다른 페이지가 보여질 수 있다.
    웹 서버에 의해서 실행되는 프로그램을 통해서 만들어진 결과물 웨에서 돌아가는 자바 프로그램

 - 웹 서버 : Nginx, Apache, IIS
    웹 서버는 클라이언트가 요청한 정적인 콘텐츠를 HTTP 프로토콜을 통하여 제공해주는 서버
    즉, 정적인 콘텐츠란 정적페이지를 말한다
    다른 역활으로는 동적인 요청이 클라이언트로 부터 들어왔을 때
    해당 요청을 웹 서버에서 처리할 수 없기 때문에 웹 컨테이너로 보내는 역할을 한다

 - 웹 컨테이너 : tomcat
    동적인 데이터들을 처리하여 정적인 페이지로 생성해주는 소프트웨어 모듈

 - 웹 애플리케이션 서버 (WAS : Web Application Server) : tomcat
    웹 애플리케이션 과 서버 환경을 만들어 동작시키는 기능을 제공하는 소프트웨어 프레임워크이다
    WAS는 동적 서버 콘텐츠를 수행하는 것이 일반적인 웹 서버와 구별되며
    주로 데이터베이스 서버와 같이 수행된다
    (웹 컨테이너 역할도 한다)

  WAS = WebServer + WebContainer
            정적페이지 + 동적페이지


 - 톰켓tomcat (WAS)
    : 포트가 8080 포트로 오라클이랑 충돌이 난다.
    환경 설정할때 포트변경 및 한글처리를 해야 된다

    -TOMCAT : 8080
    -ORACLE : 8080
    -APACHE : 80 포트
    -HTTP : 80 포트
 - 톰켓 설치
    1. https://tomcat.apache.org/ 사이트로 이동해서 다운로드하자
    2. c:\ 에 다운로드 받은 것을 이동시켜서 압축해제 하자
    3. 환경설정
        server.xml 파일 열어서 포트변경 및 한글 처리하기
        <Connector port="8090" URIEncoding="UTF-8"

        내 컴퓨터에서 마우스 오른쪽 - 속성 - 고급시스템 설정 - 환경변수 
        시스템 변수 - 새로 만들기 - 변수이름 : CATALINA_HOME, 변수값: c:\tomcat-9
        path 선택 후 편집 - 새로만들기 - %CATALINA_HOME%bin 확인
    4. 세부내용 
        bin: 명령어들이 모여 있는 곳
            실행 : startup.bat
        conf: 환경설정 
            server.xml => 포트변경 및 한글처리
        webapps: 프로젝트 파일을 넣어서 실행하면
            웝 사이트가 되는 곳
